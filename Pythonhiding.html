Data Hiding in Python

What is Data Hiding?

Data hiding is an OOP concept used to restrict direct access to the internal data of a class. It protects data from accidental modification and improves program reliability.

How Data Hiding Works in Python

Python does not enforce strict access control like other languages. Instead, it relies on naming conventions and a mechanism called name mangling.

Types of Data Members

✔ Public members
✔ Protected members (_single underscore)
✔ Private members (__double underscore)

Public Data Member


class Student:
    name = "Harsha"

s = Student()
print(s.name)
Harsha

Protected Data Member


class Student:
    _age = 20

s = Student()
print(s._age)
20

Private Data Member


class Student:
    __marks = 90

    def show(self):
        print(self.__marks)

s = Student()
s.show()
90

Why Direct Access Fails?


class Student:
    __marks = 90

s = Student()
print(s.__marks)
AttributeError

Important Points

✔ Python uses name mangling for private members
✔ __var becomes _ClassName__var internally
✔ Data hiding improves control and safety
✔ Private data should be accessed using methods
