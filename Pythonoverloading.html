Method Overloading and Overriding in Python

Method Overloading

Traditional method overloading (same method name, different parameters) is supported in languages like Java and C++. Python does not support true overloading, but similar behavior can be achieved using default or variable arguments.

Overloading using Default Arguments


class Calculator:
    def add(self, a, b=0):
        return a + b

c = Calculator()
print(c.add(10))
print(c.add(10, 20))
10
30

Overloading using Variable Arguments


class Calculator:
    def add(self, *args):
        return sum(args)

c = Calculator()
print(c.add(10, 20))
print(c.add(1, 2, 3, 4))
30
10

Method Overriding

Method overriding occurs when a child class redefines a method already defined in the parent class. Inheritance is required.

Example of Overriding


class Parent:
    def show(self):
        print("Parent class")

class Child(Parent):
    def show(self):
        print("Child class")

obj = Child()
obj.show()
Child class

Key Differences

✔ Overloading: same method name, different arguments (simulated in Python)
✔ Overriding: child class replaces parent method
✔ Overriding requires inheritance
✔ Overriding works at runtime
