Data Abstraction in Python

What is Data Abstraction?

Data abstraction is the process of hiding implementation details and exposing only the essential features of an object. It focuses on what an object does rather than how it does it.

Why Data Abstraction?

✔ Reduces complexity
✔ Improves security
✔ Enhances maintainability
✔ Provides clear interface

Abstraction in Python

Python implements abstraction using the abc module (Abstract Base Classes). An abstract class may contain abstract methods that must be implemented by child classes.

Abstract Class and Abstract Method


from abc import ABC, abstractmethod

class Shape(ABC):

    @abstractmethod
    def area(self):
        pass

Example: Data Abstraction


from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangle(Shape):
    def area(self):
        l = 5
        b = 4
        print("Area =", l * b)

obj = Rectangle()
obj.area()
Area = 20

Key Rules of Abstraction

✔ Abstract class cannot be instantiated
✔ Child class must implement abstract methods
✔ @abstractmethod decorator is mandatory
✔ Ensures consistency across subclasses

Important Points

✔ Focuses on interface, not implementation
✔ Achieved using abstract classes
✔ Supports loose coupling
✔ One of the four pillars of OOP
